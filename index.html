Introduction to Web Development

Welcome to your **Mid-Exam Preparation** material! This document covers the fundamental concepts of HTML, CSS, and JavaScript—the three core pillars of the web.

Mastering these languages is the first step toward becoming a successful developer. Let's begin the journey!

PAGE 1: HTML - The Structure

Core Principles & Rules

HTML (HyperText Markup Language) is the standard markup language for documents designed to be displayed in a web browser. It provides the **structure** of the content.

  • **Rule 1: Nesting:** Elements must be properly nested (last opened, first closed).
  • **Rule 2: Semantic Markup:** Use tags that describe the content (e.g., use `

    ` for a main heading, not just a large font size).

  • **Rule 3: Attributes:** Provide extra information about the element, always in `name="value"` pairs.

PAGE 2: HTML - Syntax & Tags

The Anatomy of an HTML Element

The basic unit is the **Element**, made up of a start tag, content, and an end tag. 

<tagname attribute="value">Content goes here.</tagname>

Key Terminology

  • **Tag:** The element name, enclosed in angle brackets (e.g., `<p>`).
  • **Attribute:** A modifier providing extra information (e.g., `href` in `<a>`).
  • **Value:** The setting for the attribute (e.g., `"https://example.com"`).

Initial Essential Tags

TagPurpose
`<!DOCTYPE html>`Declaration, defines the document type.
`<html>`Root element of the HTML page.
`<head>`Contains meta-information (not visible on the page).
`<body>`Contains all the visible page content.
`<p>`Defines a paragraph.
`<h1>` - `<h6>`Define headings (most important to least).

PAGE 3: CSS - The Style

Core Principles & Rules

CSS (Cascading Style Sheets) is used to describe the **presentation** of a document written in a markup language. It dictates colors, layouts, and fonts.

  • **Principle 1: Separation of Concerns:** Keep presentation (CSS) separate from structure (HTML).
  • **Principle 2: Cascade:** Rules are applied in a specific order (importance, specificity, source order).
  • **Principle 3: Box Model:** Every HTML element is a rectangular box, with content, padding, border, and margin.

PAGE 4: CSS - Syntax & Selectors

The Anatomy of a CSS Rule

A CSS rule consists of a **Selector** and a **Declaration Block**.

selector {
    property: value;
}
  • **Selector:** Points to the HTML element(s) you want to style (e.g., `p`, `#id-name`, `.class-name`).
  • **Property:** The aspect of the element you want to change (e.g., `color`, `font-size`).
  • **Value:** The setting you want to apply to the property (e.g., `blue`, `16px`).
  • **Declaration:** The entire `property: value;` pair.

Initial Selectors

SelectorDescriptionExample
**Type Selector**Selects all elements of a given tag name.`p { ... }`
**Class Selector**Selects all elements with a specific class attribute.`.highlight { ... }`
**ID Selector**Selects the single element with a specific ID attribute.`#header { ... }`

PAGE 6: JavaScript - The Behavior

Core Principles & Rules

JavaScript is a scripting language that enables you to create dynamically updating content, control multimedia, and animate images. It provides the **behavior** of a webpage.

  • **Principle 1: Event-Driven:** It reacts to user actions (clicks, key presses) and browser events (page load).
  • **Principle 2: Interpreted:** Code is executed line-by-line by the browser, not compiled beforehand.
  • **Principle 3: DOM Manipulation:** It interacts with the HTML structure (Document Object Model) to dynamically change content and styles.

PAGE 7: JavaScript - Syntax & Variables

Basic Syntax Rules

  • **Statements:** Instructions end with a semicolon (`;`).
  • **Case Sensitivity:** JavaScript is case-sensitive (e.g., `myVariable` is different from `MyVariable`).
  • **Comments:** Use `//` for single-line and `/* ... */` for multi-line comments.

Variables

Variables are containers for storing data values. The modern way to declare them is with `let` and `const`.

// let: a variable that can be reassigned
let message = "Hello World"; 
message = "New Message";

// const: a constant variable (cannot be reassigned)
const PI = 3.14;

**Data Types:** Initial types include **String** (text), **Number** (integers/decimals), and **Boolean** (true/false).

PAGE 8: JavaScript - Functions & DOM

Functions

A function is a block of code designed to perform a particular task, which can be reused.

function greet(name) {
    return "Hello, " + name + "!";
}

console.log(greet("Babul")); // Output: Hello, Babul!

Initial DOM Interaction

The key to dynamic web pages is interacting with the HTML elements.

// 1. Get an element by its ID
let headerElement = document.getElementById("header");

// 2. Change the element's content
headerElement.innerHTML = "New Title!"; 

// 3. Change the element's CSS style
headerElement.style.color = "red";
